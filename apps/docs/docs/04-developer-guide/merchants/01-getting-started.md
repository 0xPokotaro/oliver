# Getting Started: merchant-apiとは

merchant-apiは、商品を販売する店舗側のバックエンドシステムです。

普通のECサイトのサーバーとは異なり、人間が見る画面を一切持たない点が特徴です。

## merchant-apiの役割：x402のゲートキーパー

merchant-apiは、OliverのAIエージェントが商品を買いに来たとき、門番として振る舞います。

### Challenge：拒否する

AIエージェントが `POST /buy` で商品を購入しようとしても、最初は必ず拒否します。

HTTPステータスコード **402 Payment Required** を返し、決済署名の提示を要求します。

### Verify：検証する

AIエージェントが署名を持って再アクセスしてきたら、Facilitatorに依頼して署名を検証します。

### Serve：提供する

支払いが確認できたら、在庫を引き当て、配送手配を行い、成功レスポンス **200 OK** を返します。

## merchant-apiの特徴：ヘッドレス・コマース

### UIがない

商品画像を表示するHTML、カート画面、決済フォームを持ちません。データと決済ロジックのみを提供します。

### APIファースト

在庫データベースと直結しており、AIエージェントからのリクエストに対してJSON形式で応答します。
