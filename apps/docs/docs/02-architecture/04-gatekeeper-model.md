# Gatekeeper Model: なぜUIが不要なのか

既存のWeb APIに「決済ゲート」を設置するモデルを採用し、UI開発を不要にします。

## 設計思想

従来のECサイトでは、ユーザーが商品を選択し、カートに入れ、決済ボタンをクリックするUIが必要でした。

しかし、x402プロトコルでは、APIサーバーが直接「402 Payment Required」を返すことで、決済が必要であることを示します。これにより、専用のUIを開発する必要がなくなります。

## メリット

- **開発コストの削減**: フロントエンド開発が不要
- **標準化**: HTTP標準に準拠した統一的な決済フロー
- **AIフレンドリー**: エージェントが直接APIと対話可能
